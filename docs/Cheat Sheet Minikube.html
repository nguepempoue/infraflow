📘 Commandes Minikube (Windows + Docker Desktop)

1. Installation

# Avec Chocolatey
choco install minikube

# Avec Winget
winget install minikube

2. Démarrer le cluster

# Démarrage avec Docker comme driver
minikube start --driver=docker

# Démarrage automatique (choisit le driver dispo)
minikube start

3. Vérifier le cluster

kubectl get nodes
kubectl get pods
minikube status

4. Déployer une application simple

kubectl create deployment hello-minikube --image=k8s.gcr.io/echoserver:1.4
kubectl expose deployment hello-minikube --type=NodePort --port=8080

5. Appliquer des manifests Kubernetes

kubectl apply -f k8s/

6. Vérifier les pods dans un namespace

kubectl get pods -n infraflow

7. Accéder à un service web

# Obtenir l'URL du service
minikube service web-service -n infraflow --url

# Ou faire un port-forward
kubectl -n infraflow port-forward svc/web-service 8080:8080

8. Addons utiles

minikube addons enable ingress
minikube addons enable dashboard
minikube addons enable metrics-server

8bis. Vérifier les métriques (HPA)

# Afficher les métriques (si metrics-server est OK)
kubectl top nodes
kubectl top pods -n infraflow

# Vérifier l'état du HPA
kubectl get hpa -n infraflow

9. Monitoring (Prometheus / Grafana)

# Prometheus UI (port-forward) :
# si 9090 est déjà utilisé, remplace par 9091:9090
kubectl -n monitoring port-forward svc/monitoring-kube-prometheus-prometheus 9091:9090

# Ouvrir :
# http://127.0.0.1:9091/query
#
# Exemples de requêtes :
# up
# up{namespace="infraflow"}
# up{job="monitoring/infraflow-web-service"}

# Grafana UI (port-forward) :
kubectl -n monitoring port-forward svc/monitoring-grafana 3000:80

# Mot de passe admin Grafana :
kubectl -n monitoring get secret monitoring-grafana -o jsonpath="{.data.admin-password}" | base64 -d

# Ouvrir :
# http://localhost:3000 (user: admin)

10. Tester avec curl (optionnel)

curl http://localhost:8080
# Sur PowerShell :
Invoke-WebRequest http://localhost:8080

11. Arrêter et supprimer le cluster

minikube stop
minikube delete
